　プロジェクト課題について

プロジェクト課題では天気を表示するページを作成しました。データベースより検索を行い、必要な情報のみを抜き取り、見やすく配置するというのはとても難しいと感じました。

最も苦戦したポイントはデータベースから参照した結果を表示した後、違う検索を行った場合の前回の回答の削除です。様々な解決方法がある中で、何が一番簡潔に行うことができるかどうかを検討し、模索しました。

今回のプロジェクト課題では、検索した地点のGoogleMapのリンクを作成し、より簡単にどこにあるのかなどを知ることができるようにしました。

プロジェクト課題で作成したページ: project.html

　数当てゲーム

数当てゲームを作成しました。数当てゲームは、入力を参照し、ランダムで結果を決定し、それに習っているかどうかを検索するプログラムです。

HTMLやCSSからかけ離れたことを初めてしたため、非常に苦戦をしました。結果として、正常に動作するプログラムを作成することができました。

コンソールログや、グローバル変数を駆使することによって、変数維持を行いながら、どのようなことが起こっているのかを段階的に可視化し、結果を確認するようにすることで、不具合の解決に繋げました。

基本的に動くことがないWebサイトを簡単なゲームのようにできてしまった点には非常に驚いています。

数当てゲームのページ: kazuate.html

　各授業回の学習事項

第1回 WWW と HTML

第1回の授業では、WebページとWWW、リンクについて、HTMLの文章と要素について学びました。

HTMLには、様々なニーズに合った要素が数多と存在し、必要に応じて各自でWWWを用いて検索し、mdnなどを活用し、正しい使用方法で使用していく必要があります。

<img>　には、主な属性として src alt width height が存在し、それぞれを使用することで、自在に画像を配置することができました。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回の授業では、htmlのデザインを制御し、より見やすいWebページを作成する"CSS"に着いて学びました。

前回授業までは".html"の単体で動作していましたが、".css"の導入により、複数ファイルが連携するため、より複雑な思考が必要となりました。

演習の成果：演習2-3

第3回 JavaScript の基本

第3回授業では、JavaScriptの入門を行いました。

htmlやcssは事前に指定したものを表示することしかできませんでした。JavaScriptが増えることにより、Javaで学んだ変数や条件分岐、繰り返しの処理を実行できるようになり、できることの幅が大きく広がりました。

様々なデータベースを使用できるようになったことにより、可変的なデータを表示させることができるようになり、より外部のサイトに近しいものの作成が可能になると思いました。

演習の成果：演習3-6

第4回 DOM と JavaScript

第4回授業では、JavaScriptを用いて、テキストボックスやボタン、ボタンが押された場合の処理についてを学びました。

これらを学び、Webコンテンツの内容を読み取ったり、コンテンツに変更を加えたりする処理を行うことができるようになりました。

今後、JavaScriptを使用し、動きのあるWebページの作成を推進していこうと思います。

演習の成果：演習4-10

第5回 イベント処理

第5回授業では、JavaScriptを用いて、要素の新規作成、編集、削除する方法について学びました。

これらを実行するため、DOM(Document Object Model)を使用し、htmlには記述のない要素を新規に作成し、検索結果を表示するなどに活用することができます。

また、これまで学習したサーバーのデータベースを使用し、検索に対応した結果を表示させるまであと一歩のところまで来ました。

演習の成果：演習5-7

感想・その他

１年生ではJAVAを中心にLaTexなど様々な言語を学習しました。２年生になってからは本格的にプログラミングの学習を行い、現在奮闘中です。

今まで受けてきた授業の中ではどれもGUIの無い物でした。JAVAの実行結果はCUIで表示されるものであり、実際に視覚的に表示されるものではありませんでした。

JAVAなどで使うGUIはまだまだ難易度が高いのかなと思いながらも、成果が視覚的にわかりにくい部分はとてもありました。

今回のWebページの作成では、逐次成果が反映され、変化していくのが非常に楽しかったです。

今回のHTML,CSS,JavaScriptで１番大変だったのは、デバックでした。どのコードであってもデバックが最も大変な作業になりますが、今回のデバックではエラーを吐かない事が多々あるということに悩まされました。

大学入学以来、JAVAのお勉強をしてきて、エラーが出るのは当然という感覚でした。正しく作成されていないとコンパイルエラーを起こし、作業が１度ストップします。コンパイルを通過したとしても、実行した際にエラーを吐いて作業がストップします。それが普通になってしまっていたため、入力の環境など、様々試した後、エラーを吐かなくなったら完成のようになってしまっていました。

今回の単元では、エラーを吐かないけども実際には自分が想定していた挙動をしないとこいうことが多々あり、原因特定に多くの時間を要しました。

また、現段階でJAVAではクラスを超えた連携を行っておらず、メソッド分けが最近できるようになったところでした。今回の単元ではHTMLとCSSとJavaScriptを別ファイルを連携させる必要があるため、別ファイルで少しずつ違う書き方が必要な中で、共通の変数を使用するなど、こんがらがりそうになりました。

今回、プロジェクト課題ではデザインを凝る事が難しく、あまりうまくいかなかったため、経度緯度を使用した試みを行いました。経度緯度のデータがあるということは、マップにおいて１点を特定できるということになります。そしたら背景をマップにしてみたり、ストリートビューを表示させてみたりとできないだろうかと思いつきました。

現実はそう甘くなく、GoogleMapのリンクを設置するのが精一杯でしたが、リンクをタップすれば自動で飛んでくれる機能は非常に画期的であり、面白い案なのではないかなと個人的に思っております。

今回学習した内容を、実際に様々なところで活用ができるように、引き続き演習を続けていきたいと思います。部活動のホームページに新しい機能を取り入れてみたり、できることはたくさんあると考えています。
